Run the **Review flow** from the `dpi-system` skill to give the user an honest weekly pulse across all their DPIs (Documented Primary Interactions).

Before doing anything, read `${CLAUDE_PLUGIN_ROOT}/skills/dpi-system/references/methodology.md`.

Locate the user's DPI folder — a `dpi-<idea-name>/` with `05_scoreboard.md` and an `interviews/` folder; check the project's linked folder first, then under the user's Documents. Read the scoreboard and every interview file. Run the cross-interview synthesis: recurring patterns of stuckness, a severity read, and a deliberate "find 5 things that contradict the thesis" pass; check coverage and velocity. Update the "what could kill it" roll-up and the thesis confidence line in the files.

Then give a verdict against the bar for authentic demand (≥3 killers Derisked including indifference and will-they-pay, 0 Triggered, non-indifference ≥4 recurring, evidence behavioral not verbal), and name this week's focus: the 1–2 weakest killers and which persona layer to interview next.

If any killer has Triggered, say so plainly and propose a revised thesis version — don't paper over it.
